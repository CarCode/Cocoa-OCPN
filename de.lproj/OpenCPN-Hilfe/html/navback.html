Backup von Navigationsdaten

Backup von Navigationsdaten wurde in Version 2.5 eingeführt. Bei jedem Programmstart erfolgt ein rotierendes Backup von "navobjects". OpenCPN behält ein Backup der letzten 5 Läufe. Das kann in der opencpn.ini Konfigurationsdatei geändert werden. Dort gibt es einen Eintrag für die Anzahl der Backups mit dem Standard KeepNavobjBackup=5. Dieser Wert kann mit einem Texteditor geändert werden. Die Backups werden dann mit Dateinamen navobj.xml.[1..x] (im gleichen Verzeichnis der opencpn.ini Datei) gespeichert, wobei .1 immer das neueste Backup ist und das älteste beim nächsten Programmlauf gelöscht wird. Die beim nächsten Start geladene Datei heißt navobj.xml. Sie befindet sich unter Mac OS X im Verzeichnis ~/Library/Preferences/opencpn/, also Ihrem Home Verzeichnis. Um ein älteres Backup zu laden, beenden Sie das Programm. Suchen Sie das gewünschte Backup, benennen Sie es um zu navobj.xml und starten Sie OpenCPN neu.

Insbesondere erfahrene Anwender, die auch die Beta-Versionen von OpenCPN testen, sollten sich ihre navobj.xml Datei nochmals separat sichern, um vor Überraschungen geschützt zu sein. Es kann bei Beta-Version durchaus vorkommen, daß diese Datei zerschossen wird.

Die OpenCPN Logdatei dient ebenfalls als Backup mit ihren "LOGBOOK:" Einträgen. Sie können zur Rekonstruktion eines GPX Tracks verwendet werden.
